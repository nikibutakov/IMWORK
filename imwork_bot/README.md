## Возможности

### Для студентов:
- Онбординг с выбором курса и специализации
- Поиск вакансий по категориям
- Отклик на вакансии с резюме
- Избранные вакансии
- Карьерный центр (статьи, гайды)

### Для работодателей:
- Онбординг компании
- Пошаговое создание вакансии
- Управление своими вакансиями
- Просмотр откликов
- Статистика по вакансиям
- Тарифы (заглушка для MVP)

### Для модераторов/админов:
- Модерация вакансий (approve/reject)
- Админ-панель со статистикой
- Уведомления об ошибках

## Установка

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd imwork_bot
```

### 2. Создание виртуального окружения

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Linux/Mac
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

Или вручную установить основные пакеты:

```bash
pip install aiogram==3.x sqlalchemy==2.x python-dotenv
```

## Настройка

### 1. Получение токена бота

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/newbot`
3. Введите имя бота (отображаемое)
4. Введите username бота (должен заканчиваться на `bot`)
5. Скопируйте полученный токен

### 2. Создание файла `.env`

Создайте файл `.env` в корне проекта (`imwork_bot/.env`):

```env
# Токен бота от @BotFather (обязательно)
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz

# ID администратора для доступа к /admin_panel (получить через @userinfobot)
ADMIN_ID=123456789

# Путь к базе данных (по умолчанию imwork_bot.db)
DATABASE_PATH=imwork_bot.db

# Уровень логирования: DEBUG, INFO, WARNING, ERROR, CRITICAL
LOG_LEVEL=INFO

# Путь к файлу логов
LOG_FILE=logs/bot.log

# Максимальный размер файла лога перед ротацией (в байтах, по умолчанию 10MB)
LOG_MAX_BYTES=10485760

# Количество резервных файлов логов
LOG_BACKUP_COUNT=5
```

### 3. Получение ADMIN_ID

1. Откройте бота [@userinfobot](https://t.me/userinfobot)
2. Нажмите Start
3. Скопируйте ваш числовой ID
4. Вставьте в `.env` файл

## Запуск

### Development режим

```bash
python main.py
```

Или через Python модуль:

```bash
python -m imwork_bot.main
```

Используйте менеджер процессов, например `systemd` или `supervisor`:

## Список команд

### Пользовательские команды

| Команда | Описание |
|---------|----------|
| `/start` | Начать работу с ботом, главное меню |
| `/help` | Справка по командам (TODO V2) |

### Команды модератора

| Команда | Описание |
|---------|----------|
| `/approve <id>` | Одобрить вакансию по ID |
| `/reject <id> [причина]` | Отклонить вакансию с комментарием |

|---------|----------|
| `/admin_panel` | Открыть админ-панель (только для ADMIN_ID) |

## Архитектура

```
imwork_bot/
├── main.py              # Точка входа, dispatcher, admin panel
├── config.py            # Конфигурация, загрузка .env, логирование
├── database.py          # Подключение к БД, сессии
├── models.py            # SQLAlchemy модели (User, Vacancy, Application)
├── states.py            # FSM состояния для онбординга и форм
├── errors.py            # Middleware обработки ошибок
├── handlers/
│   ├── onboarding.py    # Онбординг пользователей
│   ├── student_jobs.py  # Поиск вакансий студентами
│   ├── employer_jobs.py # Создание вакансий работодателями
│   ├── moderation.py    # Модерация вакансий
│   └── career_center.py # Карьерный центр
├── keyboards/
│   ├── main_menu.py     # Главные меню
│   ├── employer_menu.py # Меню работодателя
│   └── job_search.py    # Клавиатуры поиска
└── logs/                # Директория для логов (создается автоматически)
```

## Чек-лист тестирования

### User Flow: Студент

#### 1. Старт и онбординг
- [ ] Отправить `/start` новому пользователю
- [ ] Проверить приветственное сообщение с выбором роли
- [ ] Нажать "👨‍🎓 Я студент"
- [ ] Выбрать курс (1-4 или магистратура)
- [ ] Выбрать специализацию
- [ ] Ввести предпочтения по стажировке (текстом)
- [ ] Проверить переход в главное меню студента

#### 2. Поиск вакансий
- [ ] В главном меню нажать "🔍 Найти стажировку"
- [ ] Выбрать категорию вакансии
- [ ] Просмотреть список вакансий
- [ ] Открыть детальную информацию о вакансии
- [ ] Нажать "Откликнуться"
- [ ] Выбрать тип отклика (быстрый/полный)
- [ ] Загрузить резюме или ввести ссылку
- [ ] Проверить подтверждение отклика

#### 3. Избранное и профиль
- [ ] Добавить вакансию в избранное
- [ ] Проверить список избранных
- [ ] Открыть профиль студента
- [ ] Проверить отображение курса и специализации

### User Flow: Работодатель

#### 1. Старт и онбординг
- [ ] Отправить `/start` новому пользователю
- [ ] Нажать "🏢 Я работодатель"
- [ ] Ввести название компании

- [ ] Проверить список откликов

- [ ] Изменить статус отклика (TODO V2)

## Известные ограничения MVP

### Не реализовано (отложено на V2):

1. **Платежи и тарифы**
   - Интеграция с платёжными системами (ЮKassa, CloudPayments)
   - Автоматическое продление тарифов
   - Ограничение функционала по тарифам

2. **Файлы и медиа**
   - Загрузка файлов резюме в облако (Google Drive, S3)
   - Хранение больших файлов
   - Предпросмотр документов

3. **Аналитика**
   - Детальная статистика просмотров вакансий
   - Конверсия воронки откликов
   - A/B тестирование

4. **CI/CD и деплой**
   - Docker-контейнеризация
   - Автоматические тесты в CI
   - Развёртывание в облаке

5. **Дополнительный функционал**
   - Чат между студентом и работодателем
   - Уведомления в email/SMS
   - Расширенный поиск с фильтрами
   - Система отзывов и рейтингов
   - Групповые рассылки

### Технические ограничения:

- База данных: SQLite (не подходит для high-load)
- Хранение сессий: MemoryStorage (сбрасывается при рестарте)
- Нет горизонтального масштабирования
- Логирование только в файлы, без агрегации

## Поддержка

2. Убедитесь, что `BOT_TOKEN` указан верно
3. Проверьте подключение к интернету
4. Убедитесь, что все зависимости установлены
